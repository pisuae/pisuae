# Project Summary
The PIS UAE Marketplace is a dynamic web application designed to facilitate the buying and selling of a diverse range of products, including electronics, clothing, and smart home devices. Operating on a commission-based vendor model, it enhances user experience with advanced analytics tools, a robust review system, and an organized product catalog. Recent improvements include a Cash on Delivery (COD) payment option and an online payment gateway via Stripe, streamlining the checkout process. The platform features an AI-driven customer support chatbot and supports Progressive Web App (PWA) functionality, allowing users to install the marketplace as a native-like app. A new user-friendly sign-up/login banner has been integrated to assist first-time customers.

# Project Module Description
The project consists of two primary modules: **Frontend** and **Backend**.

## Frontend
- **Pages**: Includes product listings, detailed product views with reviews, shopping cart, user orders, vendor signup/login, vendor dashboard, analytics, and product categories.
- **Components**: Reusable UI elements such as buttons, cards, headers, forms, a star rating system for reviews, and a customer support chatbot.
- **State Management**: Utilizes React hooks for effective state management, including a retry mechanism for API calls.
- **Checkout Page**: Supports both Cash on Delivery and Stripe payment options, requiring shipping address and phone number fields.
- **Authentication Flow**: Features a prominent sign-up/login banner for new users on the homepage, encouraging registration before purchasing.
- **CSS Updates**: The scrollbar is hidden on mobile views (screens в‰¤ 768px wide) for a cleaner user experience while maintaining scroll functionality. Smooth scrolling behavior has also been added for fluid navigation between sections.

## Backend
- **API Endpoints**: Manages CRUD operations for products, orders, vendors, reviews, and user preferences, with enhanced error handling.
- **Database Models**: Defines schemas for products, orders, user authentication, vendor profiles, and reviews, including new fields for payment methods.

# File Description Inventory
- **backend/**: Contains the backend application code, including API routes, models, services, and database migrations.
- **frontend/**: Contains the frontend application code, including pages, components (including the new ChatBot), and styles.
- **README.md**: Documentation for both frontend and backend.
- **todo.md**: Development tasks and design guidelines for vendor features.
- **image.ts**: Utility for resolving and uploading product images.
- **ProductCard.tsx**: Displays product information with average ratings and a review count.
- **ProductDetail.tsx**: Shows detailed product information along with customer reviews and a review submission form.
- **calendar.tsx**: Component for the date range picker functionality.
- **popover.tsx**: Component for the popover used in the date range picker.
- **index.html**: Main HTML file for the frontend application, updated for PWA support.
- **site.config.json**: Configuration file for the frontend, now includes Google Analytics 4 measurement ID.
- **retry.ts**: Utility for handling transient network errors with a retry mechanism, now includes a global request queue for improved resilience against DNS resolution issues.
- **user_preferences.py**: Defines the user preferences model, now includes the `avatar_url` field.
- **ChatBot.tsx**: New component for customer support, providing AI-powered responses to user inquiries.
- **Header.tsx**: Updated to include both "Sign In" and "Sign Up" buttons for better visibility.
- **Index.tsx**: Updated to add a prominent sign-up/login banner for first-time customers.
- **index.css**: Updated to hide the scrollbar on mobile views and add smooth scrolling behavior for enhanced navigation.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, Recharts, date-fns, react-day-picker
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic
- **Database**: PostgreSQL

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run `pip install -r requirements.txt`.
   - For the frontend: Navigate to `app/frontend` and run `pnpm install`.
   
2. **Build**:
   - For the frontend: Run `pnpm run build` in the `app/frontend` directory.

3. **Run**:
   - Start the backend server by executing `python main.py` in the `app/backend` directory.
   - Start the frontend application using `pnpm run dev` in the `app/frontend` directory.
