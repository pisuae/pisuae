# Project Summary
The PIS UAE Marketplace is a dynamic web application that facilitates the buying and selling of a diverse range of products, including electronics, clothing, and smart home devices. Utilizing a commission-based vendor model, the platform enhances user experience through advanced analytics, a robust review system, and an organized product catalog. Key features include Cash on Delivery (COD) payment options, an online payment gateway via Stripe, and an AI-driven customer support chatbot. The platform supports Progressive Web App (PWA) functionality, allowing users to install it as a native-like application. Recent updates have introduced a "Trending Searches" feature and a dedicated PWA Installation Guide to enhance user engagement and accessibility.

# Project Module Description
The project consists of two primary modules: **Frontend** and **Backend**.

## Frontend
- **Pages**: Product listings, detailed views, shopping cart, user orders, vendor signup/login, vendor dashboard, analytics, product categories, and a new PWA Installation Guide page.
- **Components**: Reusable UI elements including buttons, cards, headers, forms, a star rating system for reviews, a customer support chatbot, and a search bar in the sticky navigation.
- **State Management**: Utilizes React hooks for effective state management, including a retry mechanism for API calls.
- **Checkout Page**: Supports both Cash on Delivery and Stripe payment options, requiring shipping address and phone number fields.
- **Authentication Flow**: Prominent sign-up/login banner for new users on the homepage.
- **Navigation Enhancements**: Sticky mini navigation bar, smooth scrolling, floating back-to-top button, search bar, and links to the new PWA Installation Guide.
- **Search History Feature**: Saves user search queries to the database, displayed as clickable chips in the search bar.
- **Trending Searches Feature**: Displays popular search queries fetched from the backend, styled distinctively to enhance visibility.

## Backend
- **API Endpoints**: Manages CRUD operations for products, orders, vendors, reviews, user preferences, and trending searches with enhanced error handling.
- **Database Models**: Defines schemas for products, orders, user authentication, vendor profiles, reviews, and a search history table.

# File Description Inventory
- **backend/**: Contains the backend application code, including API routes, models, services, and database migrations.
- **frontend/**: Contains the frontend application code, including pages, components (including the ChatBot and search bar), and styles.
- **README.md**: Documentation for both frontend and backend.
- **todo.md**: Development tasks and design guidelines for vendor features.
- **image.ts**: Utility for resolving and uploading product images.
- **ProductCard.tsx**: Displays product information with average ratings and review counts.
- **ProductDetail.tsx**: Shows detailed product information along with customer reviews and submission forms.
- **calendar.tsx**: Component for date range picker functionality.
- **popover.tsx**: Component for the popover used in the date range picker.
- **index.html**: Main HTML file for the frontend application, updated for PWA support.
- **site.config.json**: Configuration file for the frontend, now includes Google Analytics 4 measurement ID.
- **retry.ts**: Utility for handling transient network errors with a retry mechanism.
- **user_preferences.py**: Defines the user preferences model, now includes the `avatar_url` field.
- **ChatBot.tsx**: Component for customer support, providing AI-powered responses to inquiries.
- **Header.tsx**: Updated to include "Sign In" and "Sign Up" buttons.
- **Index.tsx**: Updated to add a sticky mini navigation bar, search history functionality, and link to the PWA Guide.
- **PWAGuide.tsx**: New page providing step-by-step instructions for installing the PWA on Android and publishing it to the Google Play Store.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, Recharts, date-fns, react-day-picker
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic
- **Database**: PostgreSQL

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run `pip install -r requirements.txt`.
   - For the frontend: Navigate to `app/frontend` and run `pnpm install`.
   
2. **Build**:
   - For the frontend: Run `pnpm run build` in the `app/frontend` directory.

3. **Run**:
   - Start the backend server by executing `python main.py` in the `app/backend` directory.
   - Start the frontend application using `pnpm run dev` in the `app/frontend` directory.
