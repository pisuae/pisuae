# Project Summary
The PIS UAE Marketplace is an innovative web application designed for tech enthusiasts and consumers to buy and sell a diverse range of products, including electronics, clothing, and smart home devices. It features a user-friendly interface that enhances product exploration, offers competitive pricing, and operates on a commission-based vendor model. The platform empowers vendors with advanced analytics tools, a product review and rating system, and improved category organization to enhance the overall user experience.

# Project Module Description
The project consists of two primary modules: **Frontend** and **Backend**.

## Frontend
- **Pages**: Includes product listings, product details with reviews, shopping cart, user orders, vendor signup/login, vendor dashboard, analytics, and various product categories.
- **Components**: Reusable UI elements such as buttons, cards, headers, forms, and a star rating system for product reviews.
- **State Management**: Utilizes React hooks for effective state management and error handling through a retry mechanism for API calls.
- **Profile Page**: Updated to allow users to upload a profile picture, featuring an avatar upload overlay, file validation, and auto-saving functionality.

## Backend
- **API Endpoints**: Manages CRUD operations for products, orders, vendors, reviews, and user preferences.
- **Database Models**: Defines schemas for products, orders, user authentication, vendor profiles, reviews, and now includes an `avatar_url` field in user preferences.

# File Description Inventory
- **backend/**: Contains the backend application code, including API routes, models, services, and database migrations.
- **frontend/**: Contains the frontend application code, including pages, components, and styles.
- **README.md**: Documentation for both frontend and backend.
- **todo.md**: Development tasks and design guidelines for vendor features.
- **image.ts**: Utility for resolving and uploading product images.
- **ProductCard.tsx**: Displays product information with average ratings and a review count.
- **ProductDetail.tsx**: Shows detailed product information along with customer reviews and a review submission form.
- **calendar.tsx**: Component for the date range picker functionality.
- **popover.tsx**: Component for the popover used in the date range picker.
- **index.html**: Main HTML file for the frontend application.
- **site.config.json**: Configuration file for the frontend, now includes Google Analytics 4 measurement ID.
- **retry.ts**: Utility for handling transient network errors with a retry mechanism.
- **user_preferences.py**: Defines the user preferences model, now includes the `avatar_url` field.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, Recharts, date-fns, react-day-picker
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic
- **Database**: PostgreSQL

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run `pip install -r requirements.txt`.
   - For the frontend: Navigate to `app/frontend` and run `pnpm install`.
   
2. **Build**:
   - For the frontend: Run `pnpm run build` in the `app/frontend` directory.

3. **Run**:
   - Start the backend server by executing `python main.py` in the `app/backend` directory.
   - Start the frontend application using `pnpm run dev` in the `app/frontend` directory.
