# Project Summary
The PIS UAE Marketplace is an innovative web application designed for tech enthusiasts and consumers to easily buy and sell a wide range of products, including electronics, clothing, and smart home devices. It features a user-friendly interface that enhances product exploration and competitive pricing, operating on a commission-based vendor model. The platform provides vendors with advanced analytics tools, a comprehensive product review and rating system, and improved category organization to significantly enhance user experience. Recently, a customer support chatbot has been integrated, providing AI-powered assistance for inquiries related to products, orders, and more.

# Project Module Description
The project consists of two primary modules: **Frontend** and **Backend**.

## Frontend
- **Pages**: Includes product listings, detailed product views with reviews, shopping cart, user orders, vendor signup/login, vendor dashboard, analytics, and product categories.
- **Components**: Reusable UI elements such as buttons, cards, headers, forms, and a star rating system for product reviews. A new chatbot component has been added for customer support.
- **State Management**: Uses React hooks for effective state management, with a retry mechanism for API calls to handle errors. Non-critical API calls are now staggered and utilize a quiet retry strategy.
- **Profile Page**: Enables users to upload a profile picture, featuring an overlay for avatar uploads, file validation, and auto-saving functionality.

## Backend
- **API Endpoints**: Handles CRUD operations for products, orders, vendors, reviews, and user preferences. Enhanced error handling for non-critical endpoints has been implemented.
- **Database Models**: Defines schemas for products, orders, user authentication, vendor profiles, and reviews, with an `avatar_url` field in user preferences.

# File Description Inventory
- **backend/**: Contains the backend application code, including API routes, models, services, and database migrations.
- **frontend/**: Contains the frontend application code, including pages, components (including the new ChatBot), and styles.
- **README.md**: Documentation for both frontend and backend.
- **todo.md**: Development tasks and design guidelines for vendor features.
- **image.ts**: Utility for resolving and uploading product images.
- **ProductCard.tsx**: Displays product information with average ratings and a review count.
- **ProductDetail.tsx**: Shows detailed product information along with customer reviews and a review submission form.
- **calendar.tsx**: Component for the date range picker functionality.
- **popover.tsx**: Component for the popover used in the date range picker.
- **index.html**: Main HTML file for the frontend application.
- **site.config.json**: Configuration file for the frontend, now includes Google Analytics 4 measurement ID.
- **retry.ts**: Enhanced utility for handling transient network errors with a retry mechanism.
- **user_preferences.py**: Defines the user preferences model, now includes the `avatar_url` field.
- **ChatBot.tsx**: New component for customer support, providing AI-powered responses to user inquiries.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, Recharts, date-fns, react-day-picker
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic
- **Database**: PostgreSQL

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run `pip install -r requirements.txt`.
   - For the frontend: Navigate to `app/frontend` and run `pnpm install`.
   
2. **Build**:
   - For the frontend: Run `pnpm run build` in the `app/frontend` directory.

3. **Run**:
   - Start the backend server by executing `python main.py` in the `app/backend` directory.
   - Start the frontend application using `pnpm run dev` in the `app/frontend` directory.
