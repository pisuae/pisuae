# Project Summary
The PIS UAE Marketplace is a dynamic web application designed to facilitate seamless buying and selling of diverse products, including electronics, clothing, and smart home devices. Operating on a commission-based vendor model, it provides advanced analytics tools, a comprehensive review system, and an organized product catalog to enhance user experience. Recent enhancements include an AI-driven customer support chatbot and Progressive Web App (PWA) support, allowing users to install the marketplace as a native-like app on their devices.

# Project Module Description
The project consists of two primary modules: **Frontend** and **Backend**.

## Frontend
- **Pages**: Product listings, detailed product views with reviews, shopping cart, user orders, vendor signup/login, vendor dashboard, analytics, and product categories.
- **Components**: Reusable UI elements including buttons, cards, headers, forms, a star rating system for reviews, and a customer support chatbot.
- **State Management**: Utilizes React hooks for effective state management with a retry mechanism for API calls to handle errors. Non-critical API calls are staggered and utilize a quiet retry strategy.
- **Profile Page**: Allows users to upload a profile picture with features for file validation and auto-saving.

## Backend
- **API Endpoints**: Manages CRUD operations for products, orders, vendors, reviews, and user preferences with enhanced error handling for non-critical endpoints.
- **Database Models**: Defines schemas for products, orders, user authentication, vendor profiles, and reviews, including an `avatar_url` field in user preferences.

# File Description Inventory
- **backend/**: Contains the backend application code, including API routes, models, services, and database migrations.
- **frontend/**: Contains the frontend application code, including pages, components (including the new ChatBot), and styles.
- **README.md**: Documentation for both frontend and backend.
- **todo.md**: Development tasks and design guidelines for vendor features.
- **image.ts**: Utility for resolving and uploading product images.
- **ProductCard.tsx**: Displays product information with average ratings and a review count.
- **ProductDetail.tsx**: Shows detailed product information along with customer reviews and a review submission form.
- **calendar.tsx**: Component for the date range picker functionality.
- **popover.tsx**: Component for the popover used in the date range picker.
- **index.html**: Main HTML file for the frontend application, now updated for PWA support.
- **site.config.json**: Configuration file for the frontend, now includes Google Analytics 4 measurement ID.
- **retry.ts**: Utility for handling transient network errors with a retry mechanism, now includes a global request queue for improved resilience against DNS resolution issues.
- **user_preferences.py**: Defines the user preferences model, now includes the `avatar_url` field.
- **ChatBot.tsx**: New component for customer support, providing AI-powered responses to user inquiries.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, Recharts, date-fns, react-day-picker
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic
- **Database**: PostgreSQL

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run `pip install -r requirements.txt`.
   - For the frontend: Navigate to `app/frontend` and run `pnpm install`.
   
2. **Build**:
   - For the frontend: Run `pnpm run build` in the `app/frontend` directory.

3. **Run**:
   - Start the backend server by executing `python main.py` in the `app/backend` directory.
   - Start the frontend application using `pnpm run dev` in the `app/frontend` directory.
