# Project Summary
The PIS UAE Marketplace is a dynamic web application aimed at streamlining the buying and selling process for a diverse array of products, including electronics, clothing, and smart home devices. It operates on a commission-based vendor model, enhancing user engagement through advanced analytics, a robust review system, and a well-organized product catalog. The platform supports multiple payment methods and features AI-driven customer support to improve user interactions. Recent enhancements include real-time order tracking, an improved vendor signup process, and persistent, secured accounts for both customers and vendors.

# Project Module Description
The project consists of two primary modules: **Frontend** and **Backend**.

## Frontend
- **Pages**: Includes product listings, detailed views, shopping cart, user orders, vendor signup/login, vendor dashboard, analytics, product categories, and new pages for Saved Items and Credit Scores.
- **Components**: Reusable UI elements such as buttons, cards, headers, forms, a star rating system, a customer support chatbot, and a search bar.
- **State Management**: Utilizes React hooks for efficient state management with enhanced error handling.
- **Checkout Page**: Requires full name, email, mobile number, and shipping address for payment options.
- **Authentication Flow**: Features a sign-up/login banner on the homepage with improved error handling for network issues.
- **Navigation Enhancements**: Includes a sticky navigation bar, smooth scrolling, a back-to-top button, and links to the PWA Installation Guide, Saved Items, and Credit Scores.
- **Search History Feature**: Saves user queries as clickable chips in the search bar.
- **Trending Searches Feature**: Displays popular search queries fetched from the backend.
- **Real-Time Order Tracking**: Displays order status updates with timestamps on the Orders page.
- **Vendor Signup Form**: Updated to require Business Name, Valid Email, Mobile Number, and Verified Bank Details, with business type now stored in the description field.

## Backend
- **API Endpoints**: Manages CRUD operations for products, orders, vendors, reviews, user preferences, trending searches, and order tracking, with robust error handling.
- **Database Models**: Defines schemas for products, orders, user authentication, vendor profiles, reviews, search history, order tracking events, saved items, and credit scores. The business type is now stored in the description field.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, Recharts, date-fns, react-day-picker
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic
- **Database**: PostgreSQL

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run `pip install -r requirements.txt`.
   - For the frontend: Navigate to `app/frontend` and run `pnpm install`.
   
2. **Build**:
   - For the frontend: Run `pnpm run build` in the `app/frontend` directory.

3. **Run**:
   - Start the backend server by executing `python main.py` in the `app/backend` directory.
   - Start the frontend application using `pnpm run dev` in the `app/frontend` directory.
